eyeChat

eyeChat is a chatting application designed for people with locked-in syndrome who can only communicate via eye movement. All user interaction can be accomplished through eye blinking, though some extra features allow users with the ability to use a keyboard to communicate more quickly.

eyeChat was built as a four-day project in collaboration with Vanessa Bell. I wrote the backend as well as the majority of the logic for the user interaction.

Technologies used: HTML5/CSS3/Bootstrap/jQuery, AngularJS, Node.js, Express, Socket.io, MongoDB/Mongoose, blink-detect, headtracker.js

  • Date: August 2016
  • Source: Github

Halo

Halo is a mobile app that uses facial recognition to identify individuals while the database holds identifying and need-to-know information for emergency services personnel. This app prototype was built in 24 hours for the Hack Homelessness Hackathon in San Jose, CA. Special features include in-app emergency calling, location services, and functional facial recognition.

While this app was a group effort, I wrote all of the iOS front-end, the API calls, and the server-side routing associated with the iPhone. A MEAN front-end was built by other group members.

Technologies used: iOS/Swift, Express, Node.js, MongoDB, Kairos Facial Recognition API

  • Date: September 2016
  • Source: Github

Outdoorsy

Outdoorsy is a web application that allows users to search for and share information about outdoor areas anywhere in the world. This project was developed five weeks into the Coding Dojo program.

Technologies used: HTML5/CSS3/Bootstrap/jQuery, Python, Pylot (MVC framework developed at Coding Dojo on Flask), MySQL, Google Maps API, Open Weather Map API

  • Date: July 2016
  • Source: Github

touchSound

TouchSound is an interactive iOS application that utilizes capacitative touch sensors connected to an Arduino. The sensors act as a soundboard to manipulate the audio being played from an iPhone.

Technologies used: iOS/Swift, Express, Node.js, Arduino, Johnny-Five, Socket.io

  • Date: September 2016
  • Source: Github

Graduate Research

Vapor Transport Modeling of Continental Water Isotope Gradients

Before transitioning to software, I spent two years as a graduate student in the Terrestrial Paleoclimate group at Stanford University. While there, my focus was on understanding how the hydrologic cycle in southwestern North America responded to past warm climate regimes. As my final project as a graduate student, I worked with a reactive transport model to understand what happens to the isotopes of water vapor as a mass of air moves inland from the coast and is exposed to a number of varying environmental conditions (temperature, humidity, evapotranspiration, etc.).

Technologies used: R, NCL, HYSPLIT

  • Date: June 2016
  • Source: Github
